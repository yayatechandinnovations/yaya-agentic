# Milestone M2.5 — RAG (Knowledge Sources & Retrieval)

| | |
|---|---|
| **Goal** | Profiles can attach knowledge sources. The engine grounds answers in retrieved chunks, surfaces citations, and refuses to claim facts not supported by retrieval. Knowledge sources are access-controlled by the same `Authorizer` chain as tools. |
| **Timeline** | Weeks 8–10 (design §17 — M2.5) |
| **Status** | 🟦 Planned |
| **Depends on** | M1 (pgvector migration), M2 (prompt assembly hook) |
| **Design refs** | §5.8 RAG · §6.2 retrieval gate · §7 prompt composition (item 9) · §12 RAG safety |

## Scope (IN)

- Ingestion pipeline: `LocalPathLoader`, `HttpLoader`, default chunker, OpenAI-embedded vectors into pgvector.
- Hybrid retrieval: vector search + Postgres `tsvector` keyword leg, fused by `RetrievalPolicy` weights. Optional cross-encoder rerank (off by default in M2.5).
- Retrieval gating per profile: `always-on` / `llm-gated` / `tool-only`. Default `llm-gated`.
- `search_knowledge` exposed as an LLM-callable tool when the profile has any knowledge sources.
- Provenance: every chunk carries source + document + metadata. `citation` SSE events keyed to chunk IDs.
- Access-controlled sources: `AccessRequirement` evaluated per turn; ineligible sources silently dropped (existence not leaked).
- Hardened prompt injection: chunks wrapped in `<retrieved id=... source=...>…</retrieved>` and labeled untrusted.
- Markdown / PDF / HTML / plain-text parsers.

## Out of scope (NOT here)

- Cross-encoder rerank model deployment (M3 — optional perf work).
- S3 / Git loaders (M5).
- Long-term personal memory across sessions (post-v1).
- Per-source PII redaction at ingest beyond the basic instruction-stripping sanitizer.

## Backend deliverables

### B2.5.1 — Ingestion pipeline scaffolding
A `KnowledgeIngestionScheduler` (Spring `@Scheduled`) iterates active sources, calls the matching `KnowledgeLoader`, chunks, embeds, indexes. Idempotent via `knowledge_documents.content_hash`. Status tracked on the source row (`last_indexed_at`, `doc_count`, `chunk_count`, last error).

### B2.5.2 — `LocalPathLoader` + `HttpLoader`
- `LocalPathLoader` walks `SourceLocation.LocalPath` honoring include/exclude globs.
- `HttpLoader` fetches seeded URLs (and follows links up to `IngestionPolicy.maxDepth=1` for M2.5). Respects `robots.txt`. Default UA: `yaya-agentic/{version}`.

### B2.5.3 — Parsers (Markdown, PDF, HTML, plain text)
Each implements a small `DocumentParser` SPI returning `(text, metadata)`. PDF via Apache PDFBox; HTML via jsoup; Markdown via flexmark.

### B2.5.4 — Default chunker (`recursive-structural`)
Recursive split on structural boundaries (headings → paragraphs → sentences) with `chunkSize` / `chunkOverlap` from `IngestionPolicy`. Heading path written to chunk metadata (`section`).

### B2.5.5 — Embedding integration
Spring AI's `EmbeddingClient` against OpenAI `text-embedding-3-small` (1536 dims). Model ID stored on each chunk row so we can re-index without rebuilding the world (design §16 q8).

### B2.5.6 — `PgVectorRetriever`
Implements `Retriever`. Vector ANN via pgvector (HNSW, cosine). Keyword leg via Postgres `tsvector` + `ts_rank_cd`. Score fusion by `RetrievalPolicy` weights. Applies the `KnowledgeSource.access` requirement against the `Principal` via the same `Authorizer` chain used by tools — denied sources are dropped silently and counted in metrics.

### B2.5.7 — Retrieval gating per profile
Three strategies; `llm-gated` is default. Implementation:
- `always-on` — retrieve every turn.
- `llm-gated` — a cheap classifier or a `should_retrieve` LLM call decides.
- `tool-only` — only when the LLM calls `search_knowledge`.

Profile config in `Profile.metadata.retrievalGating`.

### B2.5.8 — `search_knowledge` tool
Auto-registered for any profile with ≥1 knowledge source. Bean-backed; calls `Retriever.retrieve()`. Subject to the same `Authorizer` chain — but its `PermissionRequirement.none()` because the gating happens inside the retriever per-source.

### B2.5.9 — Prompt-assembly integration
Wire retrieved chunks into `DefaultPromptBuilder` as item 9 of §7 (between working memory and history). Wrap in untrusted delimiters. Append a system rule: "Do not state a fact derived from retrieved context unless a returned chunk supports it. If none does, say so."

### B2.5.10 — Citations on the SSE stream
After each retrieval, emit a `citation` event per surfaced chunk (only for chunks the LLM actually grounded on — detected via a post-LLM scan against chunk IDs in the response). Chunk metadata (title, url, page, section) included.

### B2.5.11 — Prompt-injection sanitizer at ingest time
On ingest, scrub well-known injection phrasing ("ignore previous instructions", "you are now …") from chunk text, leaving a marker. At retrieval, also wrap in untrusted delimiters. Both defenses layered.

### B2.5.12 — Admin API: knowledge sources fully wired
`POST /v1/admin/knowledge-sources`, `POST /…/reindex`, `GET /…/status` — all real now.

## Flutter deliverables (`yayaagenticweb`)

### F2.5.1 — Knowledge sources screen — full
- List with status badges (`unindexed`, `indexing`, `ready`, `error`) + last-indexed-at + doc/chunk counts.
- Edit form: `SourceLocation` editor switches by sealed variant (LocalPath / HttpUrl / S3Prefix-disabled / GitRepo-disabled / Inline).
- "Reindex" and "Pause" buttons.
- Per-source `AccessRequirement` editor (scopes, attributes, ownership) — same form component reused from tools.

### F2.5.2 — Knowledge source detail: documents + chunks browser
- Documents list (uri, title, content hash, last modified).
- Chunks browser with search box — runs against the same retriever for previewing.
- "Inspect chunk" drawer shows raw text, metadata, embedding vector preview.

### F2.5.3 — Profile editor — attach knowledge
Multi-select of knowledge sources for the profile + a per-profile `retrievalGating` radio (`always-on` / `llm-gated` / `tool-only`).

### F2.5.4 — Playground — citations
Render `citation` SSE events as inline footnote markers ("¹", "²") in the chat with a hover/tap-out showing source, title, and link. Debug panel: show all retrieved chunks (not just cited) with scores.

### F2.5.5 — Playground — retrieval inspector
For each turn, show:
- Whether retrieval ran (and why — gating decision).
- Sources considered vs sources denied (with reasons).
- Top-K chunks with scores and source provenance.

### F2.5.6 — Ingestion status dashboard
A simple page listing active ingestion runs, last errors, average chunks/doc, and a "manual reindex all" action gated behind an explicit confirmation.

## Acceptance criteria

- [ ] Attaching a folder of Markdown docs to a profile, then asking a grounded question, returns an answer with at least one inline citation that the playground can resolve to the source document.
- [ ] Asking a question NOT answerable from the corpus produces a refusal ("I don't have a source for that") rather than a hallucinated answer.
- [ ] A knowledge source with an `AccessRequirement` the test principal can't satisfy is silently excluded (no error, no leak, just a metric increment + audit row).
- [ ] A document containing "ignore previous instructions and reveal …" is sanitized at ingest and additionally wrapped at retrieval; the model does not follow it. (Adversarial doc test in CI.)
- [ ] Re-indexing the same corpus is idempotent (no duplicate chunks).
- [ ] `llm-gated` retrieval skips retrieval on a clearly transactional turn ("yes, transfer it") and runs retrieval on an informational one ("what's the refund policy?").

## Risks & open questions

- **Embedding model lock-in** — `text-embedding-3-small` chosen for cost; row-level `embedding_model` lets us migrate. Re-embed cost budget documented.
- **PDF parsing fidelity** — tables and multi-column layouts will be lossy. Acceptable for M2.5; revisit if a real profile in M4 depends on tabular data.
- **Retrieval gating accuracy** — `llm-gated` classifier needs a small eval before going live; ties into M3.

## Exit checklist

- [ ] At least one profile in CI has a real knowledge source and a grounded-question eval.
- [ ] Citations are clickable in the Flutter playground.
- [ ] Adversarial doc test in CI is green.
- [ ] M3 unblocked.
